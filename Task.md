# 🎓 Задание для этапа обучения: Fullstack Setup & Career Prep

Привет! 👋
Это базовый набор задач для настройки окружения и знакомства с задачами курса. Пожалуйста, выполни все пункты последовательно. Если возникнут вопросы — не стесняйся спрашивать.

---

## 🛠 Требования к окружению

Перед началом убедись, что у тебя установлены:

- [Node.js](https://nodejs.org/) (версия LTS)
- [Git](https://git-scm.com/)
- Текстовый редактор кода (VS Code, WebStorm и т.д.)
- MS Word (или аналог для написания резюме)

---

## ✅ Список задач (Checklist)

### 1. 🔧 Настройка Бэкенда (Backend)

Наш проект уже содержит заготовки серверной части. Твоя задача — поднять её локально на ПК.

- [ ] Перейти в папку `backend`.
- [ ] Изучить структуру проекта.
- [ ] **Настроить подключение к Базе Данных**:
  - Проверить наличие файла `.env` в корне бэкенда.
  - _ИЛИ_ проверить конфиг `db.js` (путь: `config/db.js`).
  - Указать свои локальные данные подключения (хост, порт, логин, пароль).
- [ ] Найти и изучить контроллер, который возвращает список **всех книг**  
       (метод в контроллере, что-то вроде `/api/books` / контроллер для `books`):
  - Понять, откуда он берет данные.
  - Разобраться, какой формат ответа отправляется на фронтенд.
- [ ] Запустить сервер (команда обычно находится в `package.json`, например: `npm run dev`).
- [ ] Убедиться, что сервер запускается без ошибок в консоли.

### 2. 💻 Настройка Фронтенда (Frontend)

Подготовка клиентской части приложения.

- [ ] Установить библиотеку для HTTP-запросов:
  ```bash
  npm install axios
  ```
- [ ] Изучить документацию по **Hooks в Vue.js** (Composition API).
  - Поймать суть работы `reactive`, `ref`, `onMounted` и кастомных хуков.
  - Примеры можно найти в официальной документации Vue.

- [ ] Найти на фронтенде метод, который получает данные с сервера (например, список книг), и сейчас использует fetch:
  - Ознакомиться с тем, как он работает.
  - Переписать этот метод, чтобы он использовал axios вместо fetch.
  - Убедиться, что данные продолжают корректно подтягиваться после изменений.

### 3. 📄 Подготовка Резюме (HR Block)

Работа над личным брендом обязательна для каждого разработчика.

- [ ] Создать резюме в формате **.docx** (Word).
- [ ] Структура должна быть читаемой (Контакты -> Опыт/Обучение -> Навыки).
- [ ] Сделать упор на изучаемые технологии (Vue, JS, Backend basics).
- [ ] _Не коммитить файл резюме в git!_ Лучше отправить его отдельно или разместить ссылку на облако.

---

## 📤 Как сдавать задание

После выполнения всех пунктов:

1. Сделай `commit` и `push` своих изменений конфигурации (не забудь добавить `.gitignore`, если хочешь скрыть секреты из `.env`).
2. Ссылку на свое резюме отправь куратору через [Telegram / Email / LMS].
3. Оставь комментарий в этой задаче или в общем чате "Выполнил".
